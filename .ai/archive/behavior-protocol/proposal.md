# behavior-protocol：大跌行為協定內建化

## 背景動機

使用者在 `docs/PROTOCOL.md` 寫了一份「大跌行為協定」（預先承諾書）：在心平氣和時替未來恐慌中的自己預先決策，用規則對抗敘事。痛點是——**協定寫在文件裡，市場大跌、最需要它的當下，人不會主動去翻**。本 Change 讓 app 把協定「推到使用者面前」：登入後的總覽頁主動顯示「現在處於哪一個大跌情境等級、對應該做什麼／禁止什麼」，把翻文件的自律負擔改由系統承擔。

本系統無任何行情／市場 API（既定原則），因此「大盤回撤」只能以**使用者自身的投資組合資料**當 proxy 推算，這是刻意的設計取捨。

## 目標

- 新增「行為協定」頁，唯讀渲染 `docs/PROTOCOL.md` 全文，隨時可查完整規則。
- 新增「總覽／首頁」頁（登入後落地頁），顯示目前協定等級（L0 平時／L1 修正／L2 熊市／L3 深熊）與該等級的「必做／禁止」摘要，加上關鍵指標（累積 TWR／淨值／目前回撤幅度）。
- 等級由「投資組合自歷史高點的回撤」推算；門檻（−10／−20／−30%）可於設定頁調整並持久化，不寫死於程式。

## 範圍界定

### 本次做

- 行為協定唯讀展示頁（渲染 `docs/PROTOCOL.md`）。
- 總覽／首頁：等級燈號 + 必做／禁止摘要 + 關鍵指標；成為登入後預設落地頁。
- 協定等級判定邏輯（回撤 → 等級），純運算、可獨立測試。
- 等級門檻設定：設定頁可編輯、存 DB（沿用 Turso），比照現有「目標配置」的可配置模式。

### 本次不做

- 不串接任何行情／市場／新聞 API（維持系統既定原則；回撤一律以自身資料為 proxy）。
- 不做外部推播／Email／簡訊通知；「主動推到面前」僅指登入後總覽頁的顯著呈現（Streamlit app 於開啟時呈現，非背景推送）。
- 不做機動加碼金額試算與現金地板管理（PROTOCOL §2 的個人化數字不進版控、存私人筆記）；L3 只顯示規則文字，不計算加碼金額。
- 不改動既有報酬率引擎核心（TWR／淨值計算沿用，本 Change 只消費其輸出）。

## 成功標準

- 「行為協定」頁能唯讀顯示 `docs/PROTOCOL.md` 的最新內容（本機與 Cloud 部署皆可）。
- 總覽頁正確顯示：目前等級（L0–L3）、該等級必做／禁止摘要、目前自歷史高點的回撤百分比與關鍵指標。
- 回撤基準已在設計階段二選一定案並書面說明理由（候選：累積 TWR 指數回撤／排除淨投入後的淨值序列回撤）。
- 門檻可於設定頁修改、持久化，重啟後生效；不合法門檻（未維持深度遞增）被拒絕。
- 資料不足時不誤報大跌（顯示 L0 或「資料不足」而非崩壞）。
- 全套測試綠、既有行為（報酬率／配置／守門）不回歸。

## 核心業務能力

1. **協定等級判定**：由投資組合自歷史高點的回撤幅度，對應到 L0／L1／L2／L3。
2. **行為協定展示**：唯讀渲染 `docs/PROTOCOL.md` 全文。
3. **總覽儀表**：等級燈號 + 該級必做／禁止摘要 + 關鍵指標（TWR／淨值／回撤%），作為登入落地頁。
4. **門檻設定**：於設定頁編輯 L1／L2／L3 回撤門檻並持久化。

## 業務規則快照

| # | 規則 | 邊界條件 | 例外 |
|---|------|----------|------|
| BR-1 | 協定等級由「投資組合自歷史高點的回撤」判定 | 回撤基準序列於設計定案（候選：累積 TWR 指數／排除淨投入之淨值序列） | 無行情 API，以自身資料為 proxy |
| BR-2 | 門檻預設 L1 −10%、L2 −20%、L3 −30%；達門檻深度即進入該級；L0 為回撤淺於 −10% | 邊界值歸屬（剛好 −20%）之慣例於設計／SC 定案 | — |
| BR-3 | 門檻可於設定頁調整並存 DB | 須維持 0 > L1 > L2 > L3（回撤深度遞增）之合法順序 | 非法順序／非負值設定被拒絕 |
| BR-4 | 回撤以月度資料計算，等級隨每月錄入更新（非即時）；歷史高點取所有已記錄月份中基準序列的最大值 | 月度粒度為 proxy 限制 | — |
| BR-5 | 各等級「必做／禁止」摘要在 app 內以結構資料編碼；`docs/PROTOCOL.md` 為完整唯讀全文 | 結構摘要與 PROTOCOL.md 文字須人工對齊 | 屬已知維護點（兩處同步） |
| BR-6 | 資料不足（歷史月數過少或無資料）時不誤報大跌 | 顯示 L0 或「資料不足」 | — |
| BR-7 | 各等級必做／禁止內容依 PROTOCOL.md §1 表；L0＝平時（照計畫定期定額、無特別禁止，可含行為防火牆通則） | — | — |

## 外部依賴與接口契約

- `docs/PROTOCOL.md`：repo 內檔案，唯讀讀取並以 markdown 渲染（本機與 Cloud 皆以 repo 相對路徑存取）。
- 既有 `ReturnService`（累積 TWR）／`AllocationService`（淨值）：作為回撤基準的資料來源，本 Change 只消費、不改其邏輯。
- Turso（libsql）：新增「等級門檻」設定的持久化（沿用 initial-build 的連線與 Repository 模式）。

## 已知缺陷與待確認

- **回撤基準二選一**（設計階段定案並說明理由）：累積 TWR 指數回撤 vs 排除淨投入後的淨值序列回撤。
- **邊界值歸屬慣例**（設計／SC 定案）：回撤剛好等於門檻時歸較淺或較深級。
- **資料不足的等級呈現**（設計／SC 定案）：多少月數以下視為「資料不足」、如何顯示。
- **門檻設定的儲存形態**（設計定案）：新增設定表 vs 沿用既有設定機制。
