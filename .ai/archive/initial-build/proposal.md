# 資產配置管理工具 — 初版建置 (initial-build)

## 背景動機
個人投資者目前缺乏一個能「每月手動記錄各項資產金額、視覺化配置與變動、並算出真實報酬率」的私有工具。市面工具多需串接行情／券商 API，或把資料攤在第三方平台。使用者（資料工程背景、SQL 為母語）只想要：每月花幾分鐘輸入金額，就能看到資產配置佔比、總資產趨勢，以及**排除每月本金增減後的真實報酬率**。

關鍵痛點：每月會手動增減本金（如某月多投入 5 萬），裸算「期末 ÷ 期初」會把新投入的錢誤算成報酬。必須以淨投入欄位排除資本流動。

## 目標
- 提供 Streamlit 介面，輸入與輸出全部在介面完成，使用者不需手動改資料庫。
- 每月以「每個資產項目一列」的粒度記錄市值與當月淨投入。
- 視覺化資產配置（圓餅圖）與淨值變動趨勢（折線圖；淨值＝總資產 − 總負債）。
- 設定各分類目標配置比重，顯示現況對目標的偏離與再平衡提示，讓工具能「管理」而非只「記錄」配置。
- 納入負債（信用卡、貸款等）以餘額記錄，計算個人淨值；負債排除於配置佔比與報酬率之外。
- 計算排除資本流動的報酬率，同時提供 TWR（時間加權）與 MWR（金額加權/XIRR），並可拆到整體／分類／單一標的三個維度。
- 資料存於 Turso（雲端 SQLite），免費層、每月一次使用頻率下無閒置喚醒摩擦。
- 部署於 Streamlit Community Cloud（從 GitHub repo 部署），取得常駐公開 URL，免本機啟動；以 `st.login()`（Google OIDC）登入並比對 email，僅本人可進入，保護財務資料。

## 範圍界定

### 本次做
- Turso 資料表設計（持有項目主檔 + 項目 × 月份的時間序列，含市值與當月淨投入）。
- 持有項目主檔管理（新增／編輯項目，項目性質＝資產／負債，資產項目歸屬分類並記初始市值與初始成本）。
- Streamlit 月度輸入表單（新增／編輯／刪除某月各項目；資產項目記市值與淨投入、負債項目記餘額；項目從主檔選取；新月份自動帶入上月項目清單；資產分類為下拉選單）。
- 資產配置圓餅圖（選定月份各資產項目／分類佔比，不含負債）。
- 配置漂移堆疊面積圖（各分類佔比隨月份變化的趨勢）。
- 淨值變動折線圖（跨月趨勢；淨值＝總資產 − 總負債，可選疊加總資產／總負債）。
- 目標配置：設定各分類目標比重，顯示現況 vs 目標的偏離，超過門檻提示再平衡。
- 報酬率計算：TWR、MWR（XIRR）、簡單總報酬率三種＋絕對賺賠金額，維度涵蓋整體、各分類、各單一標的；未滿 12 個月只顯示期間累積報酬，不年化。
- 報酬率走勢圖：累積 TWR（或選定報酬指標）隨月份的折線。
- 報酬率計算區間可選：自開始記錄以來 / 今年以來(YTD) / 近一年 / 自訂起訖月。
- 「項目間轉移」成對輸入：一次操作自動在來源項目記負淨投入、目標項目記正淨投入，避免漏記一邊。
- 「匯出／匯入 CSV」：匯出全部紀錄為 CSV；可從 CSV 匯入還原到（新的）Turso DB。
- 登入與存取控制：`st.login()`（Streamlit 原生 OIDC，接 Google OAuth），登入後比對 `st.user.email` 僅放行本人。
- Turso 連線與 OAuth 憑證設定：本機透過 `.streamlit/secrets.toml`（納入 `.gitignore`），雲端透過 Community Cloud 後台 Secrets UI；程式一律讀 `st.secrets`。
- 部署到 Streamlit Community Cloud（GitHub repo → share.streamlit.io）。

### 本次不做
- 不串接任何行情／財報／券商 API（金額一律手動輸入）。
- 不做多幣別與匯率換算（金額一律以 TWD 輸入與儲存）。
- 不做多使用者／權限角色（個人單庫使用；登入靠 `st.login()` Google OIDC + email 比對，不自建帳密資料表）。
- 不做自動排程、通知、行動 App。
- 不做標的層級的交易明細（只記每月市值與淨投入彙總，不記逐筆買賣）。
- 不做負債的利息成本／還款明細追蹤（負債本次只記每月餘額看淨值）。

## 成功標準
- 能在 Streamlit 介面新增一個月份，系統自動帶入上月項目清單，更新市值／淨投入後存入 Turso。
- 項目以主檔穩定身分識別：跨月連乘不因名稱輸入差異而斷裂；改項目名稱不影響歷史報酬連乘。
- 圓餅圖正確反映選定月份各資產項目／分類佔比（不含負債）；折線圖正確反映跨月淨值趨勢。
- 加入一筆負債（如信用卡餘額）後，淨值＝總資產 − 總負債正確；該負債不出現在配置佔比、不進報酬率計算。
- 在含「某月額外投入本金」的情境下，報酬率不把新投入金額計為報酬：TWR 與 MWR 數值皆排除資本流動，且整體／分類／單一標的三維度皆可查得。
- 每項目可查得三種報酬率（TWR、MWR、簡單總報酬率）與絕對賺賠金額；驗證「初始成本」只影響賺賠金額與簡單總報酬率，不影響 TWR/MWR（舊持倉記錄前的價差不被算成記錄後報酬）。
- Turso 憑證不出現在版控中（本機 secrets.toml gitignore、雲端走 Secrets UI）。
- App 以 `st.login()` 強制登入，非本人 email 登入後被擋下，無法看到任何財務資料。
- 報酬率可切換計算區間（since inception / YTD / 近一年 / 自訂），數值隨區間正確重算。
- 配置漂移堆疊面積圖正確呈現各分類佔比隨月份變化；報酬率走勢圖正確呈現累積報酬折線。
- 設定各分類目標比重後，能看到現況對目標的偏離；偏離超過門檻時有再平衡提示。
- 「項目間轉移」一次輸入即在兩個項目成對記錄淨投入，兩邊金額相等方向相反。
- 「匯出 CSV」可下載完整紀錄；「匯入 CSV」可將該檔還原到空的 Turso DB，還原後資料與匯出前一致。

## 核心業務能力
1. **持有項目主檔**：維護一份持有項目清單（項目名＋所屬分類），作為單一標的報酬連乘的穩定身分來源；每個項目建立時記一次「初始市值」與「初始成本」。
2. **月度資料錄入**：以 (年月, 項目) 為粒度記錄市值與當月淨投入；項目從主檔選取；新月份自動帶入上月項目清單，只需更新數字；支援新增／編輯／刪除單列與「項目間轉移」成對輸入。
3. **負債與淨值**：負債項目每月記餘額；淨值＝總資產 − 總負債；負債排除於配置佔比與報酬率。
4. **配置視覺化**：圓餅圖（單月資產佔比）＋ 堆疊面積圖（配置佔比隨時間漂移）＋ 折線圖（淨值趨勢）＋ 報酬率走勢圖。
5. **目標配置管理**：設定各分類目標比重，顯示現況偏離與再平衡提示。
6. **報酬率計算**：三種報酬率（TWR 策略績效、MWR 個人年化、簡單總報酬率相對成本）＋ 絕對賺賠金額，三維度彙總（僅資產）；計算區間可選；未滿一年不年化。
7. **資料保全**：CSV 匯出與匯入還原（成對）。

## 業務規則快照

| # | 規則 | 邊界條件 | 例外 |
|---|------|----------|------|
| BR-1 | 每筆紀錄以 (年月, 項目ID) 為唯一鍵，記錄 `市值` 與 `當月淨投入` 兩欄；項目ID 來自持有項目主檔 | 同月同項目不可重複；項目須先存在於主檔 | — |
| BR-1b | 持有項目主檔以穩定 ID 識別項目（名稱可改不影響歷史連乘）；資產項目歸屬一個分類 | 改分類後，所有歷史月份的分類佔比皆以「當前分類」回溯重算（非時間版本化） | — |
| BR-1e | 項目有「性質」欄位：資產或負債。資產項目記市值＋淨投入（＋初始市值/成本）；負債項目每月只記餘額 | 負債不歸屬資產分類、不記初始成本/淨投入 | — |
| BR-1c | 新增月份時自動帶入上月所有項目（市值預填空待輸入、淨投入預設 0） | — | 首月無上月可帶，從主檔挑選 |
| BR-1d | 每個項目建立時記一次「初始市值」與「初始成本」；新買進項目兩者相等，舊持倉兩者可不同 | 初始市值即該項目首月 `市值` | — |
| BR-2 | `當月淨投入` 正值為投入、負值為提領；無資本流動時為 0 | — | — |
| BR-3 | TWR 月報酬率 = (期末市值 − 當月淨投入 − 期初市值) ÷ 期初市值，跨月連乘得區間 TWR；淨投入假設於月底發生（不計入當月報酬） | 期初市值為 0（首次建倉月）該月不納入連乘 | — |
| BR-4 | MWR 以 XIRR 計算：把各月淨投入視為現金流、期末市值視為終值，解年化內部報酬率 | 現金流方向需正確（投入為負、提領為正、終值為正） | 無法收斂時須有 fallback 呈現 |
| BR-4b | 未滿 12 個月的區間，TWR/MWR 只顯示期間累積報酬，不做年化 | 滿 12 個月起才顯示年化 | — |
| BR-4c | MWR/XIRR 的現金流序列：首月市值（初始市值）視為初始投入（流出）、各月淨投入為流、末月市值為終值（流入） | 首月市值不可為空 | — |
| BR-4d | 累積成本 = 初始成本 + Σ後續淨投入；賺賠金額 = 市值 − 累積成本；簡單總報酬率 = 賺賠金額 ÷ 累積成本 | 累積成本為 0 時簡單總報酬率不顯示 | — |
| BR-4e | 報酬率口徑分工【強制】：TWR/MWR 以「初始市值」起算（記錄後績效，%）；賺賠金額與簡單總報酬率以「初始成本」起算（含記錄前歷史，絕對與相對成本）。初始成本不得餵入 TWR/MWR | — | 避免記錄前價差被當成第一期報酬 |

| BR-5d | 配置漂移堆疊面積圖：呈現各資產分類佔比隨月份的變化 | 僅資產，負債不納入 | — |

## 外部依賴與接口契約
- **Turso**（雲端 SQLite）：透過 `libsql` 套件連線（舊 `libsql-client` 已被官方棄用、driver 失效，不可用）；URL 與 Auth Token 由使用者於 https://turso.tech 取得。
- **Streamlit**：UI 框架，提供表單與圖表（需 ≥ 1.42 以支援 `st.login()`）。
- **Plotly**：圖表函式庫（圓餅圖／堆疊面積／折線；Streamlit 原生不支援圓餅圖）。
- **pandas**：報酬率彙總、TWR 連乘與圖表資料整形。
- **Streamlit Community Cloud**：部署平台，從 GitHub repo 部署，提供 Secrets UI。需先有 GitHub repo。需 Streamlit ≥ 1.42（`st.login()` 原生 OIDC 支援）。
- **Google OAuth（OIDC）**：`st.login()` 的身分提供者；使用者自建 OAuth client（Client ID/Secret），redirect_uri 須含本機與雲端兩個網址。憑證放 secrets，不進 repo。
- **GitHub repo**：作為 Community Cloud 部署來源；secrets 不進 repo。
- **計算依賴**：XIRR/MWR 採 `pyxirr`（0.10.x，有 manylinux wheel，部署零摩擦；避免引入肥大的 scipy）。

## 已知缺陷與待確認
- **淨投入時點假設（已定）**：採月底發生（當月淨投入不賺取當月報酬）；對每月一次的記錄為合理簡化，誤差可接受。
- **分類／單一標的維度的 MWR 現金流歸屬**：XIRR 在子維度需以該維度自身的淨投入與期末市值計算，邊界（標的中途新增、清空、改分類）於設計階段細化。
- **「不再持有」的表示法**：賣光某項目後以「無該月份列」表示（缺列＝不持有），或記市值 0——設計階段定義並與帶入邏輯（BR-1c）、缺月處理一致。
- **缺月／不連續月份**：跨月連乘與 XIRR 對缺漏月份的處理規則，設計階段定義。
- **Google OAuth 同意畫面狀態**：停在 Testing 時 token 約 7 天過期、僅 test users 可登入；只給本人用建議將同意畫面 Publish（openid/email/profile 屬非敏感範圍，免 Google 驗證）。
- **redirect_uri 一致性**：Google OAuth client 的 redirect_uri 必須同時登記本機與雲端兩個網址，否則登入會失敗。
