# SC-047: 不合法的回撤門檻設定被拒絕，不變更既有門檻

## 行為
- **GIVEN** 回撤門檻須維持「淺→深嚴格遞增且皆為正」的合法順序：0 < L1 < L2 < L3（等價於回撤深度 −L1 淺於 −L2 淺於 −L3）
- **WHEN** 使用者在設定頁儲存不符此順序的門檻
- **THEN** 拒絕儲存、不變更已保存的門檻、提示錯誤原因

## 錯誤
下列設定一律被拒絕、不落地保存：
- L1 20%、L2 10%、L3 30%（L1 > L2，順序顛倒）
- L1 10%、L2 10%、L3 30%（L1 = L2，非嚴格遞增）
- 任一門檻為 0（如 L1 = 0）——須為正值
- 任一門檻為負值

## 邊界
- 合法設定（如 12% < 25% < 35%）→ 通過並保存
