# SC-051: 活存/定存分類新增或編輯項目時初始成本預設帶入初始市值

## 行為
- **GIVEN** 使用者在持有項目主檔新增一個資產項目，或編輯一個既有資產項目
- **AND** 該項目目前選定的分類為「活存」或「定存」
- **WHEN** 使用者在初始市值輸入框填入一個數字
- **THEN** 初始成本輸入框的值自動帶入與初始市值相同的數字（成本＝市值）
- **AND** 使用者仍可將初始成本手動改為不同於初始市值的數字，且改後的值不會被自動覆蓋
  回市值（除非使用者之後再次變動市值或分類）

## 邊界
- 僅活存/定存兩個分類比照此規則；其他分類（台股/台股ETF、美股/美股ETF、保險等）不受
  影響，成本與市值維持各自獨立輸入，改市值不影響成本欄位既有值
- 適用時機不限於新增流程：編輯既有項目時把分類由其他分類改為活存/定存亦觸發同步；
  編輯時分類本來就是活存/定存、只改市值，成本同樣跟著同步
- 使用者改選其他分類（從活存/定存切到股票/ETF 等）後，此自動同步行為即停止，之後改
  市值不再影響成本欄位
