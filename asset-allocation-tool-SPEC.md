# 資產配置管理工具 — 專案規格與決策交接

> 規劃日期：2026-06-07。本文件用於換資料夾後接續開發。
> 用法：把此檔複製到新專案資料夾，開 Claude Code 後請它讀此檔重建脈絡。

## 一、專案目標

個人用的資產配置管理工具。**每月手動輸入**每項資產金額，視覺化看出變動，並計算報酬率。**不需要串接任何行情／財報 API。**

## 二、技術選型（已定案）

```
Streamlit 介面（輸入表單 + 圖表）
        ↓ libsql-client
   Turso（雲端 SQLite，免費層）
```

- **前端/UI**：Streamlit。輸入與輸出**全部在 Streamlit 介面完成**，使用者不想手動去改資料庫或 Google Sheet。
- **資料庫**：Turso（雲端 SQLite，透過 `libsql-client`）。
  - 選它的理由：資料是表格型月度時間序列；使用者是資料工程背景、SQL 是母語；報酬率就是一句 SQL 彙總；每月一次的使用頻率下 Turso 不會有惱人的閒置暫停（Supabase 閒置 7 天要手動喚醒）。
  - 評估過但未採用：Google Sheet（不想手動編輯）、Supabase/Neon（閒置暫停 friction）、Firebase（NoSQL 不適合表格＋彙總）。
- **資料量**：很小，一年約 12 列。

## 三、功能需求

1. 月度資產金額輸入表單
2. 資產配置視覺化（圓餅圖：各項佔比）
3. 總資產變動趨勢（折線圖）
4. 報酬率計算（見下方關鍵設計）

## 四、⚠️ 關鍵設計：報酬率計算

因為使用者每月會手動增減本金（例如某月多投入 5 萬），單純比較「期末 ÷ 期初」會把**新投入的錢誤算成報酬**。

**解法：每筆紀錄分兩欄**
- `市值`：使用者輸入的當前金額
- `當月淨投入`：這月額外投入（+）或提領（−）的錢

報酬率計算要**排除資本流動**（採 TWR 時間加權 或 MWR 金額加權 概念），不能用裸的期末÷期初。

## 五、待使用者確認的問題（開發前）

1. **資料粒度**：每個資產項目一列（可看單一標的變化，預設採此），還是只記大分類一列？
2. **資產分類清單**：例如 台股ETF / 美股 / 現金 / 加密貨幣 / 債券…（尚未提供，可先用通用範例開工）
3. **幣別**：全用 TWD？還是有外幣需手動輸入匯率換算？

## 六、Turso 設定步驟（使用者本人操作，免費）

1. 到 https://turso.tech 註冊（可用 GitHub 登入）
2. 建一個 database（例如 `asset-lab`）
3. 取得 **Database URL**（`libsql://...turso.io`）與 **Auth Token**
4. 貼進本機 `.streamlit/secrets.toml`，並確認該檔已加入 `.gitignore`（Token 等於資料庫密碼，切勿 commit 或貼到對話）

1. 把此檔複製到新開發資料夾
2. 在新資料夾開 Claude Code，請它讀此檔
3. 回答第五節的三個問題
4. 開始建 App（資料表設計 → 輸入表單 → 圖表 → 報酬率計算）
